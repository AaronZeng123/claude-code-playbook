# extract-scenes

从 Claude Code 会话历史中提取应用场景，按主题聚类后生成结构化报告。

## 触发

用户输入 `/extract-scenes`，或在对话中说「提取应用场景」「分析我的 Claude Code 使用场景」。

## 前置条件

用户已运行 `extract_sessions.py`，桌面上存在 `claude-sessions/` 目录，内含 `_summary.md` 和按项目分组的会话 `.md` 文件。

## 执行流程

### 第一步：读取索引

读取 `~/Desktop/claude-sessions/_summary.md`。

这是所有会话的紧凑索引，包含：标题、日期、首条 prompt、工具调用、项目、消息数。从这里开始，不要直接读全部 .md 文件。

### 第二步：主题聚类

根据「首条 prompt + 工具组合」将索引中的会话分组，形成 15-25 个候选分类。

聚类方法：
- 提取每个会话的「首条 prompt 主题词」（前 50 字中的关键动词和名词）
- 提取每个会话的「特征工具组合」（如 Read+Edit+Write = 代码修改，Bash+Agent = 自动化，Skill+Write = 内容创作）
- 将主题词相似 + 工具组合相似的分到一组
- 每组起一个描述性的临时名称（如「代码调试与重构」「文档内容创作」「数据分析与报表」）

### 第三步：采样深读

对每个候选分类，随机采样 3-5 个会话（优先选消息数多的），读取其完整 `.md` 文件，确认：
1. 分类名称是否准确（会不会误归类）
2. 该分类下的典型工作流是什么
3. 是否有需要拆分或合并的分类

验证后调整：
- 样本太分散的分类 → 拆分为 2-3 个子类
- 样本同质化高的分类 → 合并
- 最终控制在 12-15 个大类

### 第四步：生成报告

输出 `~/Desktop/claude-sessions/scenes-report.md`，格式如下：

```markdown
# Claude Code 应用场景报告

**生成时间**：YYYY-MM-DD HH:MM
**会话总数**：N
**场景分类数**：M

---

## 场景 1：场景名称

**会话数**：X
**典型用户**：角色描述（如后端开发、产品经理、创业者）

**使用模式**：
- 做什么：一句话概括这个场景下用户用 Claude Code 做什么
- 怎么做：2-3 句话描述典型工作流
- 省了什么：和不用 AI 相比节省的时间/步骤

**高频工具**：Tool1、Tool2、Tool3

**典型案例**：
- 「会话标题 1」— 一句话说明做了什么事
- 「会话标题 2」— 一句话说明做了什么事
- 「会话标题 3」— 一句话说明做了什么事

---

## 场景 2：...

（重复以上结构）

---

## 统计摘要

| 场景 | 会话数 | 占比 |
|------|--------|------|
| ... | ... | ... |

---

*由 Claude Code extract-scenes skill 生成*
```

### 第五步：总结

在对话框给用户一个简短总结：共 M 个场景分类，TOP 3 高频场景是什么，报告保存在哪里。

## 注意事项

- 不要读全部 .md 文件，先用 `_summary.md` 做聚类
- 同一会话可能属于多个场景，选最主要的那个归入
- 标题不要泛化（不要用「代码工作」这种），要具体（如「Bug 修复与代码审查」）
- 场景名称使用用户视角（「我用 Claude Code 做 XXX」而不是「Claude Code 的 XXX 功能」）
- 报告中不要出现 session_id，只保留会话标题
