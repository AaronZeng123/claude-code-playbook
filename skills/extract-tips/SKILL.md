# extract-tips

从 Claude Code 会话历史中提取操作技巧、CLI 命令、配置方案和 workflow 模式。

## 触发

用户输入 `/extract-tips`，或在对话中说「提取操作技巧」「总结 Claude Code 使用技巧」。

## 前置条件

用户已运行 `extract_sessions.py`，桌面上存在 `claude-sessions/` 目录，内含 `_summary.md` 和按项目分组的会话 `.md` 文件。

## 执行流程

### 第一步：筛选候选会话

读取 `~/Desktop/claude-sessions/_summary.md`，用以下规则筛选可能包含技巧的会话：

**标题/首条 prompt 包含关键词**：
- 「怎么」「如何」「快捷键」「配置」「设置」「技巧」「最佳实践」
- 「CLI」「命令行」「终端」「Bash」
- 「skill」「MCP」「hook」「插件」
- 「报错」「失败」「不生效」「bug」

**工具组合特征**：
- Bash 调用次数 >= 5 且含 `claude` 或 `mcp` 相关命令
- 使用了 Skill 工具
- 消息数 >= 20（长对话通常有探索和试错过程）

取候选会话 TOP N（控制在 30-50 个），优先选消息数多、工具种类丰富的。

### 第二步：深读提取

逐个读取候选会话的完整 `.md` 文件，提取以下内容：

1. **CLI 命令**：`claude` 开头的命令、`mcp` 相关命令、管道组合
2. **配置方案**：CLAUDE.md 配置、settings.json 修改、MCP server 配置、hooks 设置
3. **快捷键与操作**：/ 命令、键盘操作、UI 交互技巧
4. **Workflow 模式**：多步骤串联的典型工作流（如「先用 A 工具做 X，再用 B 工具做 Y」）
5. **工具组合套路**：高频工具搭配（如 Bash+Read+Edit 的代码修改模式）
6. **报错排坑**：从会话中提取解决问题的方法（错误现象 → 排查过程 → 解决方案）

每条技巧记录来源会话标题。

### 第三步：归类整理

将提取的技巧归入以下类别：

- **效率命令**：CLI 命令、快捷键、/ 命令
- **配置调优**：CLAUDE.md、settings.json、MCP、hooks
- **Workflow**：多步骤串联的操作模式
- **工具使用**：特定工具（Bash、Agent、Skill 等）的使用技巧
- **避坑指南**：常见错误和解决方案
- **隐藏功能**：不那么明显但有用的功能

### 第四步：生成报告

输出 `~/Desktop/claude-sessions/tips-report.md`，格式如下：

```markdown
# Claude Code 操作技巧报告

**生成时间**：YYYY-MM-DD HH:MM
**来源会话数**：N

---

## 效率命令

### 技巧标题
- **命令**：`具体命令`
- **作用**：一句话说明
- **来源**：「会话标题」

（同类技巧间用空行分隔）

---

## 配置调优

### 技巧标题
- **配置**：配置内容和位置
- **作用**：一句话说明
- **来源**：「会话标题」

---

## Workflow

### 技巧标题
- **步骤**：1 → 2 → 3
- **适用场景**：什么时候用
- **来源**：「会话标题」

---

## 工具使用

### 技巧标题
- **工具**：工具名称
- **用法**：具体操作方式
- **来源**：「会话标题」

---

## 避坑指南

### 问题现象
- **原因**：为什么会出这个问题
- **解决**：具体解决步骤
- **来源**：「会话标题」

---

## 隐藏功能

### 技巧标题
- **发现**：功能描述
- **用法**：如何使用
- **来源**：「会话标题」
```

### 第五步：总结

在对话框给用户总结：提取了多少条技巧，分别来自哪些类别，报告保存在哪里。

## 注意事项

- 技巧要可操作，不要泛泛的「善用 XXX」
- 每条技巧标注来源会话（只用标题，不用 session_id）
- 命令和配置直接给可复制版本
- 如果某个类别没有内容就跳过，不要硬凑
- 避免重复：同一条技巧如果在多个会话中出现，合并为一条，标注多个来源
